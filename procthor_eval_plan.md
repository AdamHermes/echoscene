# ProcTHOR Evaluation Plan: Walkability Metric

This document outlines the step-by-step plan to evaluate the output of `echoscene` (`output/released_full_model`) in the **ProcTHOR** (AI2-THOR) environment, starting specifically with the **Walkability (Navigability)** metric.

## 1. Objective
Calculate the Walkability score for scenes generated by EchoScene. 
* **Definition of Walkability**: The proportion of the floor space that an agent can safely navigate without colliding with furniture or architectural boundaries.
* **Mechanism**: Using ProcTHOR's 2D NavMesh (Navigation Mesh) projection.

## 2. Prerequisites
1. **AI2-THOR/ProcTHOR Installation**: Ensure `ai2thor` and `procthor` packages are installed.
2. **EchoScene Output**: The JSON/NPZ scene files located in `output/released_full_model` which contain bounding boxes, object categories, and room boundaries.

## 3. Implementation Steps

### Step 1: Data Conversion (EchoScene to ProcTHOR JSON)
ProcTHOR requires scenes to be defined in a specific JSON format containing rooms, floor polygons, and object instances.
* **Script required**: `scripts/procthor_eval/convert_echoscene_to_procthor.py`
* **Tasks**:
  * Parse the room layout (walls, floor plan) from the EchoScene output and convert it into ProcTHOR's `polygon` format.
  * Map EchoScene object categories to ProcTHOR asset IDs (e.g., `chair` -> `Chair_1`, `bed` -> `Bed_1`).
  * Extract 3D bounding box coordinates (position, rotation, scale) and format them into the ProcTHOR object schema.

### Step 2: ProcTHOR Environment Initialization
Set up the AI2-THOR controller to load the generated custom JSON scenes.
* **Script required**: `scripts/procthor_eval/eval_walkability.py`
* **Tasks**:
  * Initialize the `ai2thor.controller.Controller` instance.
  * Load the converted scene JSON into the controller using the `CreateHouse` action.

### Step 3: NavMesh Generation & Walkability Calculation
Once the scene is loaded, AI2-THOR's physics engine drops the objects and builds the NavMesh.
* **Tasks**:
  * Call `controller.step(action="GetReachablePositions")` to extract all points on the 2D floor grid that an agent can stand on.
  * Calculate the total valid floor area of the empty room (without furniture).
  * Calculate the available reachable area (with furniture placed by EchoScene).
  * **Metric Formula**: 
    `Walkability Score = (Number of Reachable Positions) / (Expected Reachable Positions for an optimal/ground-truth room)` 
    *(Alternatively, simple percentage of floor covered by NavMesh vs occupied by objects).*
  * **Validation**: Check for isolated NavMesh islands (e.g., a chair completely blocking a door to another part of the room) to penalize severe blockages.

## 4. Proposed Code Structure

```python
# Pseudo-code for eval_walkability.py

from ai2thor.controller import Controller
import json

def calculate_walkability(scene_json_path):
    # 1. Start ProcTHOR Controller
    controller = Controller(
        agentMode="default",
        visibilityDistance=1.5,
        scene="ProcTHOR", # Use ProcTHOR procedural scene mode
        gridSize=0.25
    )
    
    # 2. Load the converted EchoScene house
    with open(scene_json_path, 'r') as f:
        house_data = json.load(f)
    
    controller.step(action="CreateHouse", house=house_data)
    
    # 3. Get total floor area (Optional: calculate purely from bounding box)
    # ...
    
    # 4. Get reachable positions via NavMesh
    event = controller.step(action="GetReachablePositions")
    reachable_positions = event.metadata["actionReturn"]
    
    # Calculate score based on the number of reachable points / grid size
    walkable_area = len(reachable_positions) * (0.25 ** 2) 
    
    controller.stop()
    return walkable_area
```
